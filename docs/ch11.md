# 第十一章：红线与石碑

他深吸一口气，开始重建完整的算法模型。

首先，他必须确认排序的主键。

前八个人的数据：

| 序号 | 姓名 | 性别 | 年龄 | 座位 | 身份 |
|------|------|------|------|------|------|
| 1 | 陈国辉 | 男 | 52 | A3-7 | 地产商 |
| 2 | 李秀芬 | 女 | 25 | C9-2 | 沈氏集团安全工程师 |
| 3 | 张伟 | 男 | 28 | B6-5 | 富二代 |
| 4 | 王建国 | 男 | 56 | A3-9 | 官员 |
| 5 | 刘志强 | 男 | 45 | B2-4 | 律师 |
| 6 | 林美凤 | 女 | 49 | A5-6 | 全职太太 |
| 7 | 叶婉秋 | 女 | 41 | C1-8 | 教师 |
| 8 | 周建明 | 男 | 44 | B4-3 | 医生 |

他仔细审视这个表格，试图找出隐藏的模式。

陈国辉——52岁，A3-7。基准。
李秀芬——25岁，C9-2。
张伟——28岁，B6-5。
王建国——56岁，A3-9。
刘志强——45岁，B2-4。
林美凤——49岁，A5-6。
叶婉秋——41岁，C1-8。
周建明——44岁，B4-3。

他和沈策的数据呢？

肖山，31岁，B5-7。程序员。
沈策，年龄未知，B5-8。神秘男人。

如果排序主键是"年龄"，那基准是52。左右子数组的划分是以52为界。

但这个模型在前六个人之后就被打破了——肖山（31岁）被标记为第七个"候选者"，但实际第七个是叶婉秋（41岁）。

为什么会跳过31岁的肖山，选了41岁的叶婉秋？

难道算法有另一种排序的维度——比如座位号的远近？

A3-7（陈国辉）作为基准。按位置"距离"排序：

C9-2、B6-5、A3-9、B2-4、A5-6、C1-8、B4-3——

这不像是按简单的一维距离排序。

但如果按二维欧几里得距离呢？

将区域映射为数字：A=1，B=2，C=3。

陈国辉的位置向量：(1, 3, 7)。

到其他人的"距离"：

李秀芬 (3, 9, 2)：sqrt((1-3)^2 + (3-9)^2 + (7-2)^2) = sqrt(4+36+25) = sqrt(65) ≈ 8.06
张伟 (2, 6, 5)：sqrt((1-2)^2 + (3-6)^2 + (7-5)^2) = sqrt(1+9+4) = sqrt(14) ≈ 3.74
王建国 (1, 3, 9)：sqrt(0+0+4) = 2
刘志强 (2, 2, 4)：sqrt(1+1+9) = sqrt(11) ≈ 3.32
林美凤 (1, 5, 6)：sqrt(0+4+1) = sqrt(5) ≈ 2.24
叶婉秋 (3, 1, 8)：sqrt(4+4+1) = sqrt(9) = 3
周建明 (2, 4, 3)：sqrt(1+1+16) = sqrt(18) ≈ 4.24

按距离从近到远排序：王建国(2) → 林美凤(2.24) → 叶婉秋(3) → 刘志强(3.32) → 张伟(3.74) → 周建明(4.24) → 李秀芬(8.06)

点名顺序：李秀芬(8.06) → 张伟(3.74) → 王建国(2) → 刘志强(3.32) → 林美凤(2.24) → 叶婉秋(3) → 周建明(4.24)

不是从近到远，也不是从远到近。

但——等一下。如果看距离排列，王建国(2)是最远的...不对，最近的。但实际上他是第四个被点的...

等等——他突然注意到了另一个规律。

如果把基准设置为A3-7，然后按位置维度分别排序——先排区域（A、B、C）、再排排数、再排座位号——会得到什么结果？

陈国辉的座位是A-3-7。这可以视为一个数字137（编码为A=1）。

其他座位编码：
李秀芬：C-9-2 = 392
张伟：B-6-5 = 265
王建国：A-3-9 = 139
刘志强：B-2-4 = 224
林美凤：A-5-6 = 156
叶婉秋：C-1-8 = 318
周建明：B-4-3 = 243

排序：
137（陈国辉）→ 139（王建国）→ 156（林美凤）→ 224（刘志强）→ 243（周建明）→ 265（张伟）→ 318（叶婉秋）→ 392（李秀芬）

点名顺序：1 陈国辉(137) → 2 李秀芬(392) → 3 张伟(265) → 4 王建国(139) → 5 刘志强(224) → 6 林美凤(156) → 7 叶婉秋(318) → 8 周建明(243)

在排序后的序列中，陈国辉(137)是基准（最小值）。
第一次分割：左右子数组以137为基准。
"大于137"的元素都进入右子数组。

但是——点名顺序不是按排序后的序列顺序，而是一种交错。

基准(137) → 最大(392) → 次大(265) → 次小(139) → 中偏小(224) → 中偏大(156) → 更大(318) → 更小(243)

这不是快速排序，也不是任何一种标准的排序算法。

这看起来像是——像是某种自定义的"从两端向中间"的遍历？

不是。从数据点的顺序看：

137(1) → 392(8) → 265(6) → 139(2) → 224(4) → 156(3) → 318(7) → 243(5)

1→8→6→2→4→3→7→5

这个序列没有任何明显的数学规律。

但等等——如果看每次点名的"方向"呢？

基准是A区3排7座（A3-7）。

李秀芬：C区9排2座（C9-2）——最远、最角落、完全相反的方向。
张伟：B区6排5座（B6-5）——中心偏左的位置。
王建国：A区3排9座（A3-9）——与基准同排、相邻两座。
刘志强：B区2排4座（B2-4）——靠近前排中心。
林美凤：A区5排6座（A5-6）——与基准同区、靠后两排。
叶婉秋：C区1排8座（C1-8）——C区前排、偏右。
周建明：B区4排3座（B4-3）——中心区、偏左的位置。

他在脑海中画出宴会厅的座位图，从A到C，从第1排到第10排。

然后把他自己和沈策也标上去。

肖山：B区5排7座（B5-7）。
沈策：B区5排8座（B5-8）。

他突然注意到一个细节。

肖山——B5-7。沈策——B5-8。

他们和王建国（A3-9）、刘志强（B2-4）的位置都比较靠近中心。而陈国辉（A3-7）也在中心区域。

但李秀芬（C9-2）在最角落，叶婉秋（C1-8）在另一个角落。

当肖山的目光扫过"李秀芬"这三个字的时候，他的思绪短暂地偏离了算法。他想起了她——那个戴黑框眼镜的短发女人，被点名时甚至没有尖叫。当厉鬼念出她的名字时，他已经隐约捕捉到了座位编码和年龄之间的某种关联模式。他几乎是本能地脱口而出："C9-2——那个哈希值不在你的主键索引里！"

那声喊叫让厉鬼的判断出现了一瞬间的延迟。

她的身体已经开始融化——从指尖开始，皮肤裂解成细小的数据碎片——但在肖山喊出那句话的瞬间，融化停在了半途中。她的右臂和半边脸颊留下了永久性的伤痕——不是血肉的伤，而是数据化的痕迹，像是显示器的坏点区域，皮肤表面呈现出像素化的碎裂纹路。

她是第一个被点名却没有死的人。

她活下来了——代价是她永远带着那道数据化的疤痕。

如果把整个宴会厅看作一个二维平面，每个被点名的人都是在平面上的一个点。这些点之间，除了距离基准的远近外，是否还有另一种关系——

比如，构建一个二维KD树？

KD树是一种分割k维空间的数据结构。在KD树中，每一层选择一个维度作为分割轴，将空间分为两部分，然后递归地分割子空间。

如果在二维KD树中，第一层按区域（A、B、C）分割，第二层按排数分割，第三层按座位号分割——

或者第一层按排数分割？

肖山揉了揉太阳穴。他的大脑已经快要超负荷运转了。

但他觉得KD树的思路是对的。如果用KD树来组织所有人的座位数据，那么点名的顺序就是从树根开始，深度优先遍历，每个节点被访问时就杀死这个人。

但KD树是中序遍历——左子树、根节点、右子树——这个顺序和他观察到的点名顺序不符。

而且KD树是结构化的数据组织方式，不是排序算法。

不过，快速排序本身就是一种"二叉排序树的构建过程"——每次分割都会产生一个基准节点，左右子数组形成左右子树。

他需要回到快速排序的基本逻辑上。

也许他要从另一个角度来看——不是看"谁死了"，而是看"谁还活着"。

活着的人中，哪些人已经被排除了（即不在任何子数组中需要进一步处理），哪些人还在等待被排序？

他重新观察宴会厅。

A区的人已经死了三个（陈国辉、王建国、林美凤）。A区还有人活着。
B区的人死了两个（张伟、刘志强、周建明是B区的）。B区还有很多人。
C区——叶婉秋死了。李秀芬还活着——她是被点名却没有被算法'提交'的例外，和肖山一样。C区也还有很多人。

如果按区域分组——A区死了3个，B区死了3个，C区死了2个。

好像没有明显的区域偏差。

他又看向沈策。

沈策在B5-8，他们在同一排相邻座位。

肖山突然冒出一个念头——

算法会不会是根据"与我的相对位置"来排序的？

他很快否定了这个想法。这太以自我为中心了，厉鬼不可能围绕着他来设计的。

除非——他是关键。他是木偶师选中的人。他的存在本身就是这个游戏的核心。

**倒计时：10。**

只剩十秒了。

他盯着屏幕上的厉鬼。厉鬼的头部再次旋转，像雷达一样扫过人群。

它会点第九个人的名字。

他必须预测出第九个人。

根据他的模型——无论是什么排序算法——第九个人都是可以预测的。

他拿出全部的计算力，在脑海中飞速构建。

如果基准是陈国辉(A3-7)，按这个"基准位置"分割空间。李秀芬(C9-2)是最远的，所以先被点。然后是近一些的张伟(B6-5)。

然后算法回到基准最近的两个方向：

一个在基准的同排（王建国，A3-9）。
一个在基准的前方（刘志强，B2-4）。

然后是比刘志强更近但不同位置的林美凤(A5-6)。

然后是不同方向的叶婉秋(C1-8)和更快中心的周建明(B4-3)。

这像是一个螺旋结构——从基准开始，向外螺旋扩散，但不是简单的螺旋，而是一种按照某种顺序——

**6。**

他猛地捕捉到脑海中闪过的灵感。

这不是基于位置的排序。

这是基于"人"本身的某种属性的排序。位置、年龄、姓名——这些都只是标签。真正的排序键值，是"被邀请的原因"。

他快速回顾每个人的身份：

陈国辉——地产商，是主办方的重要合作伙伴。
李秀芬——沈氏集团安全工程师，IT部门骨干。
张伟——富二代，他父亲是主办方的投资方。
王建国——官员，是主办方在政府的靠山。
刘志强——律师，主办方的法律顾问。
林美凤——全职太太，她丈夫是主办方的合伙人。
叶婉秋——教师，她是被邀请的某位嘉宾的女儿的老师。
周建明——医生，主办方的家庭医生。

这些身份——在经济链上的位置——从高到低，或者从低到高——这可能是排序的键值！

如果按"在主办方的社会关系网中的重要性"排序——

陈国辉（地产商，重要合作伙伴）是基准。
李秀芬（安全工程师，技术骨干）被第二个点名。
张伟（富二代，关系户）也靠前。
王建国（官员，靠山）重要。
刘志强（律师，法律顾问）中等重要。
林美凤（全职太太，亲属关系）中等。
叶婉秋（教师，边缘关系）边缘。
周建明（医生，私人关系）私密。

但这是——主观判断。不是算法。如果是算法，它必须有明确的排序标准。

**3。**

没有时间了。

他最后看了一眼屏幕上的厉鬼。它正在微笑——如果那种嘴型叫做微笑的话。

然后他注意到了。

厉鬼的每次点名，它的手指都会指向一个方向。而那个方向——

陈国辉——指向A区。
李秀芬——指向C区。
张伟——指向B区。
王建国——指向A区。
刘志强——指向B区。
林美凤——指向A区。
叶婉秋——指向C区。
周建明——指向B区。

区域的序列：A → C → B → A → B → A → C → B。

这看起来像是某种三进制计数器或者循环序列。

但如果从另一个角度看——A、B、C三个区域轮流出现，但不是简单的ABCABC循环。

而且，这个序列怎么对应到快速排序的区域划分？

**2。**

**1。**

**0。**

倒计时归零。

屏幕上，厉鬼的手指向了——

B区。

"第——九——位——"

"赵——德——胜。"

B区响起一声惨叫。肖山扭头看去，只见一个五大三粗的男人正抱着头蹲在地上，全身颤抖。

"不——不——不——求求你——我女儿——"

他的身体开始融化。

但肖山没有看他的死亡过程。他在飞速地记录数据：

赵德胜，男，约50岁，B区某排某座。

他没有看清具体座位，但这是一个重要的数据点。

B区。第九个人。

按照刚才的A → C → B → A → B → A → C → B → B 的序列，这次是两个B连续。

如果把这个序列和排序算法联系起来——

等等——如果他不是按区域，而是按排数呢？

陈国辉：3排
李秀芬：9排
张伟：6排
王建国：3排
刘志强：2排
林美凤：5排
叶婉秋：1排
周建明：4排

排数序列：3→9→6→3→2→5→1→4

最近到来的排数是——赵德胜在B区，大概在B区的后排，可能是8排或9排左右。

所以排数序列：3→9→6→3→2→5→1→4→(?)

没有明显的规律。

但突然——肖山想到了一个完全不同的角度。

快速排序的核心是"分治"。每次选择一个基准，将问题一分为二。

但这里的"分治"不是按年龄、不是按座位——而是按一种肖山一直忽略的东西。

**死亡顺序本身。**

陈国辉第一个死。
李秀芬第二个被点名。
张伟第三个死。

但张伟的年龄28在所有人的年龄中是最小的。他是"最极端"的那个。

如果排序不是对所有人排序，而是对"死人序列"进行排序？

不，这太诡异了。

等一下——他重新审视前几个人的年龄和座位号的数值。

陈国辉：52岁，7座。
李秀芬：25岁，2座。
张伟：28岁，5座。
王建国：56岁，9座。
刘志强：45岁，4座。
林美凤：49岁，6座。
叶婉秋：41岁，8座。
周建明：44岁，3座。

座位号序列：7→2→5→9→4→6→8→3

如果按座位号排序：2,3,4,5,6,7,8,9。
点名顺序：7,2,5,9,4,6,8,3。

在升序序列中，7排第6位，被第一个点。2排第1位，被第二个点。5排第4位，被第三个点。

这确实像是快速排序的过程！

座位号升序排列：2,3,4,5,6,7,8,9

第一次分割，以7为基准（第一个被点的陈国辉的座位号）：
左子数组（小于7）：2,3,4,5,6
右子数组（大于7）：8,9

右子数组按某种顺序：8（叶婉秋），9（王建国）。
但点名中王建国（9）在第四位被点，叶婉秋（8）在第七位被点。

所以右子数组是：9（第四位），8（第七位）。

左子数组按某种顺序：2（李秀芬，第二位），5（张伟，第三位），4（刘志强，第五位），6（林美凤，第六位），3（周建明，第八位）。

左子数组以2为基准（第二个被点）？还是以5为基准？

不对——他需要重新思考。

如果把座位号的序列看作是快速排序的benchmark，基准是第一个被点的陈国辉的座位号7。

右子数组（大于7）是&#123;8,9&#125;。从大到小是9（王建国，4th）然后8（叶婉秋，7th）。
但叶婉秋在第7位，中间隔了刘志强（5th）和林美凤（6th）。

这说明左子数组和右子数组不是交替被点的。而是——快速排序的分治递归——先将整个数组分成两个子数组，然后递归处理每个子数组。

但在递归中，右子数组&#123;8,9&#125;有两个元素。它需要被进一步排序。

&#123;8,9&#125;中选基准：按什么规则？中位数？第一个？最后一个？

如果按第一个——9（王建国的座位号）作为子基准。那右子数组中只有一个比9小的元素——8（叶婉秋的座位号）。

所以在从大到小的递归中：
右子数组&#123;8,9&#125;以9为基准：9被点（4th），然后8被点（7th）。

但叶婉秋是在7th被点的，中间隔了5th（刘志强）和6th（林美凤）。

为什么？因为算法此时切换到左子数组进行进一步处理？

左子数组&#123;2,3,4,5,6&#125;，以2为子基准（第二个被点的李秀芬的座位号）。

&#123;大于2&#125;的子数组：&#123;3,4,5,6&#125;
这个子数组以5为基准（第三个被点的张伟的座位号）：&#123;小于5&#125;的子数组&#123;3,4&#125;，&#123;大于5&#125;的子数组&#123;6&#125;。

&#123;大于5&#125;的子数组&#123;6&#125;只有一个元素——林美凤（6th）。
&#123;小于5&#125;的子数组&#123;3,4&#125;——以3（周建明）为基准？但周建明是8th被点的，远在叶婉秋之后。

如果说左子数组的下一层先处理&#123;大于5&#125;的子数组，再处理&#123;小于5&#125;的子数组，那顺序应该是：

1. 全局基准7（陈国辉）→右子数组&#123;8,9&#125; 
2. 右子数组以9为基准（王建国，4th）
3. 右子数组剩下的8（叶婉秋，7th）

所以如果&#123;小于5&#125;的子数组&#123;3,4&#125;先处理&#123;大于3&#125;的子数组&#123;4&#125;（刘志强，5th），再处理&#123;小于3&#125;的子数组{}（空），然后以3为基准处理&#123;3&#125;这个元素...

不，这太牵强了。他在强行套用快速排序的递归过程，但每一步都需要自由选择方向（先左还是先右），这样的模型可以拟合任何数据。

他需要更严格的验证。

如果把所有已知数据点——姓名、年龄、座位、职业、死亡顺序、被点名时的方位——全部输入一个抽象的"算法模拟器"中。

他反复运行不同的排序算法：

冒泡排序：相邻比较，交换。不匹配。

插入排序：依次插入有序序列。不匹配。

选择排序：每次选择最大值。前三位（52,25,28）不是简单的选择最大或最小。

合并排序：二分后排序再合并。可能性较大，但需要验证。

快速排序：选基准后分割。最有可能。

他重新构建快速排序的完整过程，这次严格按照标准算法：

标准快速排序（升序）：
1. 选择第一个元素为基准。
2. 从两端向中间遍历，将小于基准的放左边，大于基准的放右边。
3. 递归地在左右子数组中重复。
4. 每一次"选中"一个元素作为基准，并进行分割操作。

如果每次都返回基准被选择的"位置"，那点名的顺序就是基准被选择的顺序。

也就是说：第一个基准（陈国辉，52）→分割数组→在左子数组中选择第二个基准→在右子数组中选择第三个基准→继续递归。

快速排序中选择基准的方式通常是最左边的元素。但这里——被点名的顺序 = 基准被选择的顺序。

标准快速排序每次递归选择一个基准，然后分别对左右子数组进行递归。

基准选择顺序（如果按最左边）：先选全局最左（陈国辉），然后递归到左子数组选最左（可能是某个年龄小于52的人），再递归到右子数组选最左...

这不是标准的"先深度遍历左子树"或"先深度遍历右子树"的顺序。如果是中序遍历：左→根→右，或者前序：根→左→右，后序：左→右→根。

如果按中序遍历——根（陈国辉）→左子树→右子树——那排序顺序应该是：
根（陈国辉）→左子树的根→左子树的递归→右子树的根→右子树的递归。

但点名顺序是：根（陈国辉）→右子树（李秀芬）→左子树（张伟）。

所以是先右后左？

前序遍历（根→左→右）是：陈国辉、左子树根、左子树的左...
后序遍历（左→右→根）是：左子树最深的左叶子、...

不匹配。

张伟(28)是左边界。
王建国(56)是右边界的新值。

前八个人的哈希值大小顺序应该和他们的点名顺序一致。但前八个人的哈希值是什么？他不可能知道哈希函数。

姓名笔画最多的？笔画最少的？都不是。

等等——如果基准不是按任何属性选的，而是——**陈国辉是碑上的第一个人**。

七个人——刻在碑上的七个人——是十年前建造这栋楼时死去的工人。

陈国辉——他姓陈。死去的工人中，会不会也有一个姓陈的？

这是一个通道。活人的死亡，是为了唤醒碑上的亡灵。每个被点名杀死的人，都是在"匹配"一个碑上的名字——用同样的姓、或者同样的生辰、或者某种对应的关系。

肖山感到一阵寒意从脚底升起。

如果每个被点名杀死的人都是在"对应"一个碑上的死者——

那七次点名，对应七个碑上的名字。

已经点了九个人。但——叶婉秋和周建明的名字不在前七个之内。

所以不是九个对应七个。而是——所有被点的人最终都会对应到七个灵魂上。

但有多少人会被点？一直到宴会厅里所有人都死光？还是只有特定的七个？

他的大脑高速运转。快速排序——算法的每一次分割都选择一个基准、杀死一个人。但基准的数量远远不止七个。

在三百人的数据集中，快速排序的平均递归深度是log2(300) ≈ 8.2层，每层选择多个基准。总选择基准的次数等于数据集大小——三百次。

如果每选一个基准就杀一个人——那宴会中的所有人都会死。

这不是七个人。这是——所有人。

肖山倒吸一口冷气。

这个规则怪谈——从一开始就不是"死七个人就停止"的游戏。

这是——**一场针对所有人的屠杀。**

用快速排序算法驱动的群体性屠杀。

他——作为被选中的破局者——要在三百人被全部杀死之前，找到算法的漏洞。

他的目光落在脖子上那条若隐若现的红线上。

木偶师说——他能给肖山的，只有时间。

而时间——现在以每60秒一次点名的方式在流逝。

每人60秒。三百人。五个小时。

他需要在五个小时内破解这个算法。

或者——找到那个能让算法提前终止的"终止条件"。

在快速排序中，终止条件是"子数组只包含一个元素"。

当排序完成时，所有元素都是独立的子数组，算法终止。

但那个状态——是所有人的死亡。

他不能等到那个时候。

他要找到——在排序完成之前——提前逃脱的方法。

肖山闭上眼睛，不再看屏幕上的倒计时。

他开始在大脑中构建一个三百人的数据集。

三百个人的名字、年龄、职业、座位号。

他要运行一遍完整的快速排序。

不是按任何预设的键值——而是——随机？不——不是随机——是按照厉鬼选择的规则。

但规则是什么？

他找不到。

除非——他的大脑突然闪现出一个可怕的可能——

规则就是**没有固定规则**。

点名不是基于任何人类可理解的键值排序。而是一种——对于人类来说，完全随机的——超自然力量的选择。

如果是这样，那他的所有建模和分析都是徒劳的。

"不。"肖山在黑暗中握紧拳头,"一定有什么规律。一定有。"

他的目光落在大厅穹顶那些垂下的线上。

线。所有线。

"所有人都是提线木偶。"

他的脑海中响起木偶师的话。

"提线木偶的核心——是线的另一端。"

他猛地看向舞台。

厉鬼在屏幕上。但牵动每个人脖子的线的另一端——不是屏幕，而是天花板。

在天花板上方——在宴会厅的地基和舞台之间——一定有一个"线控中枢"。

那个地方——

就是石碑所在的位置。

而石碑上的七个名字——就是解开所有谜题的关键。

他转向沈策，压低声音说：

"我需要去舞台下面。"

沈策的眼睛在黑暗中微微发亮。

"你知道那意味着什么吗？"

"知道。"

"倒计时还有——43秒。你只有43秒。"

"够了。"

肖山深吸一口气，在黑暗中迈出了第一步。

他朝着舞台的方向走去。

每一步，脖子上的红线就紧一分。

但他没有停下脚步。

因为他知道——在这个被算法驱动的恐怖游戏中，唯一的生路，就是成为算法的终结者。

他——肖山——一个写了十年代码的程序员——要和一头鬼——比赛谁的算法更高效。

舞台的灯光在黑暗中忽明忽暗，像是在嘲笑他的不自量力。

但他不在乎。

他走到舞台前，蹲下身，用手摸索着地板上的那道印记。

那道印记——像是被血浸透的木头——在黑暗中散发着微弱的红光。

他用指甲抠了抠地板的缝隙。

木板松动了一下。

他的心跳加速了。

下面——真的有空间。

他用力撬开那块木板。

在木板下方——是一个凹陷的空间。

里面躺着一块——长满了青苔的石碑。

碑上刻着七个名字。

肖山瞪大了眼睛。

那七个名字中，第一个——赫然写着：

**——肖山——**

不。

不可能。

那座碑——是十年前刻的。十年前，他还不认识这个城市里的任何人。

但他的名字——怎么会出现在这里？

碑上的七个名字：

肖山。
李秀芬。
王建国。
周建明。
赵德胜。
叶婉秋。
陈国辉。

七个名字——有五个已经被点名、已经死了。

而活着的有两个——他，和李秀芬。

她是第一个被点名却没有死的人。不是算法的仁慈——是肖山的喊声在最后一刻让厉鬼的判断产生了逻辑冲突。

**他和她，都是例外。**

不，不对——如果他是碑上的人，他应该是以死者的身份出现在碑上的。但他还活着。

——除非。

——他不是十年前死去的工人。

——他的名字是在某个时间点——被后来加上去的。

他猛地想起一个小时前——在那座古墙下——木偶师在他的后颈上种下了一条红线。

那时——木偶师在他的脖子上，用血写下了什么。

**他的名字。**

木偶师把"肖山"两个字，写在了他的后颈上。

然后，把这两个字"刻"在了石碑上。

他的命——是从木偶师那里借来的。

"我来告诉你为什么你没有死。"

沈策的声音突然在他身后响起。

"因为你还不是死人。"

"你是活人——你是一个被写在墓碑上的活人。"

"你的名字在碑上，但你的命还在自己手里。"

"厉鬼不能杀你——因为你的名字还没有被它的算法真正'读取'到。"

"它只能显示你的名字。但杀不了你。"

"因为你是——这场算法的——BUG。"

肖山感到全身的血液都在倒流。

他——一个活人——一个程序员——成为了一个超自然排序算法中的BUG。

而BUG——

要么被修复（杀死）。
要么——成为新的规则。

他看着石碑上的七个名字。

六个已死。一个活着。

第六个名字的下面——

还有一行小字。

那是用一种几乎看不清的字体刻下的：

"当第七人看见此碑，规则将改写。"

"代价——提线者之命。"

肖山缓缓站起身，看向屏幕上的厉鬼。

屏幕上的倒计时——

**10。**

九秒后，第十一次点名。

厉鬼的头部缓缓转向他。

"第——十——一——位——"

他的嘴型——在说出一个名字——

"江——"

但声音没有出来。

屏幕上的数字突然停止了跳动。

**10——**

它停住了。

整个大厅陷入了一种诡异的寂静。

连厉鬼——都停住了。

肖山低头看着石碑上那行小字。

"当第七人看见此碑，规则将改写。"

他——作为第七个人——看见了碑。

规则——正在改写。

屏幕上的厉鬼开始疯狂地抖动、变形、扭曲。

它的身体一会儿膨胀、一会儿缩小，像是在被某种力量反复撕扯。

"不——不——不——"

厉鬼终于发出了人类的声音——不再是低沉的嘶吼，而是一种——充满恐惧的尖叫。

"你不能这么做——你不能——"

肖山盯着屏幕，看着厉鬼的形态在崩溃、瓦解。

"规则——被改写了——"

厉鬼的声音越来越小，越来越远，像是在被一股力量拖入深渊。

然后——屏幕上的猩红色光芒突然熄灭。

整个宴会厅陷入了一片黑暗。

但这一次——黑暗只持续了三秒。

灯光重新亮起。

水晶吊灯发出柔和的光芒，照亮了整个大厅。

地毯上的血迹消失了。破碎的桌椅复原了。那些死去的宾客——他们的座位上摆着鲜花，像是在悼念。

所有人——都消失了。

整个大厅——只剩下肖山和沈策两个人。

舞台上的屏幕——变成了一块普通的LED屏幕，显示着主办方的宣传片。

石碑——静静地躺在舞台下方的凹槽里。

肖山站在那里，浑身是汗，脖子上还缠着那条断了一半的红线。

他看着沈策。

沈策看着他。

他们同时——看向倒在地上的一块东西。

那是从屏幕上掉下来的——半截血红色的——人皮。

厉鬼——消失了。

或者说——它被规则反噬了。

肖山缓缓蹲下身，捡起地上的人皮。

皮上刻着一行字：

"排名不分先后，死亡不分彼此——感谢你，救了我。"

字迹——和石碑上的最后一句话——一模一样。

这是木偶师留下的信。

或者说——这是那个被囚禁在厉鬼体内的、真正的第七个人——留下的遗言。

肖山闭上眼睛，深深地吸了一口气。

结束了。

这场点名游戏——结束了。

他赢了。

但代价是什么——那行小字上说的"提线者之命"——他自己的命？还是木偶师的命？

他不知道。

但此刻——这座死寂的宴会厅中——他第一次感到——活着。

他活下来了。

那些被算法夺走生命的人——陈国辉、张伟、王建国、刘志强——他们的名字还停留在他的脑海中，像一个永远调试不完的程序。

他欠他们一条命。

不——他欠他们一个答案。

他要找出这个规则怪谈的根源——为什么是他们？为什么是这座宴会厅？为什么——是他的名字，写在石碑上？

他睁开眼睛，在明亮的灯光下——第一次真正地——看清了石碑上那七个名字的全部内容。

在"肖山"这两个字的后面——还有一行更细的字：

"提线之偶——生于1994年3月15日——卒于——（空白）——"

那是他的出生年月日。

但卒年——是空白的。

因为他还没有死。

因为他——是这场游戏中——唯一的赢家。

也是唯一的——输出。

在算法结束的瞬间，他不再是一个数据点。

他——成为了规则本身。
